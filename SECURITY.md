# Security Policy

## Supported Versions

This project is currently in early stage. Security fixes are provided on the latest `main` branch.

## Reporting a Vulnerability

Please do **not** open public issues for sensitive vulnerabilities.

Use one of the following private channels:
- GitHub private vulnerability reporting (preferred)
- Contact maintainer: `conanxin@users.noreply.github.com`

Please include:
1. Affected file/module
2. Reproduction steps
3. Expected impact
4. Suggested mitigation (optional)

## Scope Notes

- This tool can call external URLs and optional LLM APIs.
- Do not hardcode API keys in source code or screenshots.
- Avoid committing local evidence files and environment artifacts.
